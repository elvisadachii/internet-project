Transmission Control Protocol / Internet Protocol

The foundational suite of communication protocols that defines how data is transmitted across the Internet.

Definition

TCP/IP (Transmission Control Protocol / Internet Protocol) is the core set of communication rules and standards that govern how data is packaged, addressed, transmitted, routed, and received across networked computers. Often called the "language of the Internet," TCP/IP is the reason a device in Nairobi can seamlessly communicate with a server in California.

TCP/IP is not a single protocol but a suite of protocols — a conceptual framework consisting of four abstraction layers, each responsible for a specific aspect of network communication.

The Two Core Protocols

Internet Protocol (IP) is responsible for addressing and routing. Every device on the Internet is assigned an IP address (e.g., 192.168.1.1 for IPv4, or a longer hexadecimal address for IPv6). IP breaks data into packets and routes each packet independently through the network toward the destination.

Transmission Control Protocol (TCP) operates on top of IP and ensures reliable, ordered, and error-checked delivery. TCP establishes a connection via a "three-way handshake" (SYN → SYN-ACK → ACK), transmits data in segments, and guarantees that all segments arrive correctly and in order.

The Four Layers

  • Application Layer — Handles high-level protocols such as HTTP, FTP, DNS, and SMTP. This is where user-facing software operates.
  • Transport Layer — TCP and UDP operate here. This layer manages end-to-end communication, error detection, and data ordering.
  • Internet Layer — IP operates here, handling logical addressing and routing of packets across multiple networks.
  • Network Access Layer — Deals with physical hardware, MAC addresses, and the transmission of raw data over physical media (Ethernet, Wi-Fi).

TCP vs UDP

While TCP guarantees reliable delivery, UDP (User Datagram Protocol) trades reliability for speed. UDP is used in applications where some packet loss is acceptable — such as live video streaming, online gaming, and VoIP calls — because low latency matters more than perfect delivery.

TCP/IP was adopted as the standard protocol of ARPANET in 1983, marking the birth of the modern Internet.

IPv4 vs IPv6

IPv4 uses 32-bit addresses, supporting approximately 4.3 billion unique addresses. As the Internet grew, this proved insufficient. IPv6 uses 128-bit addresses, providing an astronomically large address space (approximately 3.4 × 10³⁸ addresses) to support the ongoing growth of connected devices worldwide.
