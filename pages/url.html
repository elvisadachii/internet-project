Uniform Resource Locators

The standardised addressing system that uniquely identifies every resource — page, image, video, or file — on the World Wide Web.

Definition

A Uniform Resource Locator (URL) is a reference (address) to a resource on the Internet. It specifies the location of a resource on a computer network and the mechanism for retrieving it. Every web page, image, video, and file accessible on the Internet has a unique URL.

URLs are a subset of URIs (Uniform Resource Identifiers), a broader standard. Tim Berners-Lee introduced URLs as part of his original web design to give every resource a unique, navigable address.

Anatomy of a URL

A complete URL has several distinct components:

  • Scheme/Protocol — Specifies how the resource is accessed: https://, http://, ftp://, mailto:
  • Subdomain — An optional prefix, commonly www, that identifies a specific section of a site.
  • Domain Name — The human-readable name of the server, e.g., example.com.
  • Top-Level Domain (TLD) — The extension: .com, .org, .ke, .edu.
  • Port — Optional, e.g., :8080. Defaults to 80 for HTTP, 443 for HTTPS.
  • Path — Points to a specific resource on the server, e.g., /blog/article.
  • Query String — Optional parameters after ?, e.g., ?page=2&sort=date.
  • Fragment — A #anchor link to a specific section within the page.
Example: https://www.example.co.ke/products?category=tech#reviews
Scheme: https | Subdomain: www | Domain: example.co.ke | Path: /products | Query: category=tech | Fragment: #reviews
Introduced by
Tim Berners-Lee, 1994
Standard Body
IETF (RFC 3986)
Max Length
2,048 characters (IE limit)
Secure Scheme
HTTPS (encrypted)

URL vs URI vs URN

A URI (Uniform Resource Identifier) is the broadest term — it identifies a resource. A URL identifies a resource by its location and access method. A URN (Uniform Resource Name) identifies a resource by name, independent of location (e.g., ISBN numbers for books). All URLs are URIs, but not all URIs are URLs.

URL Encoding

URLs can only contain certain characters. Special characters and spaces are encoded using percent-encoding (e.g., a space becomes %20, & becomes %26). This ensures URLs are transmitted correctly across all systems.
