Hypertext Transfer Protocol

The application-layer protocol that governs how web browsers and servers communicate — the backbone of data exchange on the World Wide Web.

Definition

Hypertext Transfer Protocol (HTTP) is an application-layer protocol used for transmitting hypermedia documents, such as HTML. It defines the rules and format by which messages are sent and received between web browsers (clients) and web servers. HTTP is the foundation of data communication on the World Wide Web.

HTTP follows a request–response model: the client sends a request to the server, and the server returns a response. It is a stateless protocol, meaning each request is independent — the server does not retain information about previous requests by default (cookies and sessions address this limitation).

HTTP Methods

HTTP defines a set of request methods (verbs) indicating the desired action:

  • GET — Retrieves data from the server (e.g., loading a web page). Should not change server state.
  • POST — Submits data to the server (e.g., form submissions, creating resources).
  • PUT — Updates an existing resource entirely.
  • PATCH — Partially updates a resource.
  • DELETE — Removes a specified resource.
  • HEAD — Same as GET but returns only headers, not body.
  • OPTIONS — Describes communication options for the target resource.

HTTP Status Codes

Every HTTP response includes a three-digit status code indicating the outcome:

  • 1xx Informational — Request received, processing continues.
  • 2xx Success — 200 OK (request succeeded), 201 Created.
  • 3xx Redirection — 301 Moved Permanently, 302 Found.
  • 4xx Client Error — 400 Bad Request, 403 Forbidden, 404 Not Found.
  • 5xx Server Error — 500 Internal Server Error, 503 Service Unavailable.

HTTPS — Secure HTTP

HTTPS (HTTP Secure) adds a layer of encryption using TLS (Transport Layer Security). This encrypts the data exchanged between the browser and server, protecting against eavesdropping and man-in-the-middle attacks. HTTPS is now standard for all websites, especially those handling sensitive information.

HTTP/2 (2015) introduced multiplexing, allowing multiple requests over a single connection. HTTP/3 (2022) runs on QUIC (over UDP) for even lower latency.

HTTP Versions

HTTP/1.0 opened a new connection for every request. HTTP/1.1 introduced persistent connections and chunked transfers. HTTP/2 enabled multiplexing and header compression. HTTP/3 uses the QUIC protocol over UDP for dramatically reduced latency, especially on mobile networks.
