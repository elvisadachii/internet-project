Domain Name System

The Internet's distributed directory service that translates human-readable domain names into machine-readable IP addresses.

Definition

The Domain Name System (DNS) is a hierarchical, decentralised naming system for computers, services, and other resources connected to the Internet or a private network. Often described as "the phonebook of the Internet," DNS translates memorable domain names (like google.com) into numerical IP addresses (like 142.250.80.46) that computers use to locate each other.

Without DNS, users would have to memorise complex numerical IP addresses to visit websites. DNS makes the Internet user-friendly and navigable.

The DNS Resolution Process

  1. Browser Cache Check — The browser first checks its own cached DNS records.
  2. OS Cache Check — If not cached, the operating system checks its local DNS cache and the hosts file.
  3. Recursive Resolver — The query goes to the ISP's DNS resolver (a recursive resolver), which handles the query on behalf of the client.
  4. Root Name Server — If the resolver lacks the answer, it queries one of 13 root name server clusters worldwide.
  5. TLD Name Server — The root server directs the resolver to the appropriate TLD server (e.g., the .com server).
  6. Authoritative Name Server — The TLD server points to the domain's own authoritative server, which returns the actual IP address.
  7. Response — The IP address is returned to the browser, cached, and used to initiate a connection.

DNS Record Types

  • A Record — Maps a domain to an IPv4 address.
  • AAAA Record — Maps a domain to an IPv6 address.
  • CNAME Record — Aliases one domain name to another (e.g., www → bare domain).
  • MX Record — Specifies the mail servers for the domain.
  • TXT Record — Stores text information, often used for verification and spam prevention (SPF, DKIM).
  • NS Record — Indicates which name servers are authoritative for the domain.
The entire DNS system resolves billions of queries per day in milliseconds, forming one of the Internet's most critical infrastructure components.

DNS Security

DNSSEC (DNS Security Extensions) adds cryptographic signatures to DNS records to protect against DNS spoofing and cache poisoning attacks. DNS over HTTPS (DoH) and DNS over TLS (DoT) encrypt DNS queries to prevent eavesdropping, improving user privacy.
